📦 Box Sizing

Você definiu width: 300px em ambas as caixas. Mas o tamanho final é diferente!

🔴 content-box

Padrão do CSS (sem definir box-sizing)

width: 300px
padding: 20px
border: 10px
📐 Como o CSS calcula:
width: 300px (só o conteúdo)
+ padding: 20px + 20px (esquerda e direita)
+ border: 10px + 10px (esquerda e direita)
─────────────────────
Tamanho real: 360px 😱
Azul = área do conteúdo (300px)
Vermelho = borda (10px cada lado)
O padding empurra pra fora da área azul

🟢 border-box

box-sizing: border-box

width: 300px
padding: 20px
border: 10px
📐 Como o CSS calcula:
width: 300px (tudo junto)
- padding: 20px + 20px (cabe dentro)
- border: 10px + 10px (cabe dentro)
─────────────────────
Tamanho real: 300px ✅
Verde = tudo cabe nos 300px
Vermelho = borda (já inclusa)
O que você define é o que você tem
💡 Resumo:
Com content-box, o width define só o conteúdo. O padding e a borda são somados por fora, o que faz a caixa crescer além do que você esperava.

Com border-box, o width define o tamanho total da caixa. Padding e borda já estão incluídos. O que você escreve é o que aparece na tela.

Por isso, a maioria dos projetos começa com:
* { box-sizing: border-box; }
Pra garantir que todos os elementos se comportem de forma previsível. ✅
